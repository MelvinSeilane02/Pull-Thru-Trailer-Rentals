Trailer UI Card Structure (Top Picks & Listings)
1️⃣ Image section (top – 40%)
- Large, clean trailer image
- Same aspect ratio for all cards
- Optional corner tag (top-left):

    - “Most Popular”
    - “Available Today”
- Tag uses yellow background
Why:
- Visual trust
- Fast recognition
- Stops scrolling

2️⃣ Card body (middle – 45%)
Trailer name (bold)
- Example: Standard Utility Trailer
Primary use case (small text)
- “Ideal for furniture & garden waste”

Key specs (icon row)
- 📏 Size (e.g. 6ft × 4ft)
- ⚖️ Max load (e.g. up to 750kg)
Keep this to 2–3 icons max.

3️⃣ Price & availability (bottom – 15%)
Price
- “From R350 / day”
- Larger font than specs

Availability indicator
● Green: Available
● Yellow: Limited
● Red: Fully booked

Dot + text (never color alone).

4️⃣ Action area (always visible)
- Primary button:
    - Book Now (yellow background)
-   Secondary (optional)
    - “View details”
One primary action only.

🧠 Visual hierarchy (very important)
Top → Image
Middle → Understanding
Bottom → Decision

If users scroll and only see:

- Image
- Name
- Price
- Book button

You’ve already won.
